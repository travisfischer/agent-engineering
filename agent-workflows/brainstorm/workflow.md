# Brainstorm Workflow

## Goal

Answer what to build before planning how to build it.

## Hard Constraints

1. Brainstorm runs are documentation-only.
2. Do not implement code during brainstorm execution.
3. A brainstorm run is incomplete until a brainstorm artifact is written.

## Flow

1. Assess whether requirements are already clear.
2. Ask focused clarification questions one at a time.
3. Propose 2-3 approaches with pros, cons, and recommendation.
4. Capture key decisions, open questions, and next steps.
5. Stop after handoff options unless the user explicitly asks to continue.
