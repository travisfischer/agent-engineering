# Plan Workflow

## Goal

Create a requirement document that is specific enough to implement and specific enough to review.

## Hard Constraints

1. Plan runs are documentation-only.
2. A run is incomplete until one requirement file is written.

## Flow

1. Resolve planning input from user request and any brainstorm artifact.
2. Clarify only what is missing.
3. Research local patterns and prior decisions.
4. Use external research only when risk or uncertainty justifies it.
5. Choose planning depth.
6. Draft the requirement document.
7. Run the checklist.
8. Present handoff.
