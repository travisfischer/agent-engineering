# Planning

This skill defines how to produce a requirement or specification document after brainstorming.

## Hard Constraints

1. Planning runs are documentation-only.
2. Do not implement code or runtime behavior while running this skill.
3. Always persist a planning artifact before ending the workflow.

## Core Process

1. Resolve input from user request and prior brainstorms.
2. Research local context first.
3. Use external research only when justified.
4. Choose planning depth.
5. Write a requirement document with testable scope and acceptance criteria.
6. Run the planning checklist.
