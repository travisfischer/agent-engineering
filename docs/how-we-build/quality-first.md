# Quality First

Optimize for correctness, reliability, maintainability, and trustworthiness before speed or cost shortcuts.

## Core Principle

Every change should improve overall system quality or clearly preserve it.

## Commitments

1. No corner-cutting
2. Design quality before implementation speed
3. Verification is part of the work
4. Quality across all artifacts
5. Continuous quality improvement

## Testing and Evaluation Expectations

1. Use a balanced testing strategy.
2. Increase rigor with user impact and blast radius.
3. Use explicit evals for LLM-powered features.
4. Use deterministic and non-deterministic quality mechanisms where appropriate.
5. Any escaped defect should produce stronger guardrails.

## Review Standards

Review meaningful changes for:

1. Design fit
2. Functional correctness
3. Simplicity and maintainability
4. Test and eval adequacy
5. Security and failure handling
6. Operational impact
7. Documentation and ergonomics
