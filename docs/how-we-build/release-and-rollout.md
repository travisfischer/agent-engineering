# Release and Rollout

Changes should only be rolled out when there is clear confidence in quality and behavior.

## Commitments

1. Quality-first release confidence
2. Goldilocks rollout scope
3. Compound engineering feedback
4. Agent-native operability

## Minimal Release Checklist

1. The change is scoped and reviewed.
2. Required checks passed.
3. Key behavior can be observed after release.
4. A rollback or disable path is clear.
5. The team is confident the change improves the system.
