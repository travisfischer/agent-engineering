# Agent-Native Development

Development environments, tooling, and workflows should be designed so agents can execute meaningful engineering work with high reliability and low friction.

## Constitutional Commitments

1. Developer-action parity for agents
2. Shared workspace model
3. Composable development primitives
4. Observable execution
5. Fast feedback loops

## Planning and Build Rules

1. When adding a workflow, define how an agent executes it.
2. Avoid tooling choices that require human-only steps for routine work.
3. Prefer text-first and scriptable tools.
4. Standardize repeatable workflows in prompts, scripts, and docs.
5. Capture reusable guidance from completed work.
